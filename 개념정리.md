OutSystems 자격증 시험(Associate Reactive Developer 등)을 준비하신다면, 변수의 **Scope(범위)**와 **Lifecycle(생명 주기)**을 구분하는 것이 가장 중요합니다. 시험에 자주 출제되는 핵심 변수 5가지와 사용 예시를 정리해 드릴게요.

---

### 1. Local Variable (지역 변수)

* **정의:** 특정 Screen이나 Action 내부에서만 사용되는 변수입니다.
* **특징:** 해당 Screen이 닫히거나 Action이 종료되면 메모리에서 사라집니다.
* **시험 포인트:** "특정 로직 안에서만 임시로 데이터를 저장할 때 무엇을 쓰는가?"
* **사용 예시:** * 화면에서 검색창에 입력한 검색어 저장.
* 반복문(For Each) 내에서 합계를 계산하기 위한 임시 변수.



### 2. Input / Output Parameter (입력/출력 매개변수)

* **정의:** Action 간에 데이터를 주고받기 위한 통로입니다.
* **특징:**
* **Input:** 호출하는 쪽에서 값을 넣어줌.
* **Output:** 호출받은 로직이 결과를 담아서 돌려줌 (앞서 질문하신 '안팎 모두 사용 가능'의 주인공).


* **시험 포인트:** "Action 간 데이터를 전달하는 방법", "Output Parameter의 가시성(Visibility)"
* **사용 예시:**
* `GetCustomerDetail`이라는 Server Action에 `CustomerId`(Input)를 넣고, `CustomerData`(Output)를 받아오는 경우.



### 3. Client Variable (클라이언트 변수)

* **정의:** 사용자의 브라우저(로컬 저장소)에 저장되는 변수입니다.
* **특징:** * 기본 데이터 타입(Integer, Text 등)만 저장 가능하며, **Record나 List는 저장할 수 없습니다.** (시험 단골 문제)
* 앱을 새로고침하거나 껐다 켜도 값이 유지됩니다.


* **시험 포인트:** "로그인한 사용자의 UI 설정 유지", "저장 가능한 데이터 타입의 제한".
* **사용 예시:** * 다크모드/라이트모드 설정값 유지.
* 리스트 화면에서 사용자가 마지막으로 선택한 정렬 기준 저장.



### 4. Site Property (사이트 프로퍼티)

* **정의:** 서버 측에 저장되는 전역 변수(Global Variable)입니다.
* **특징:**
* 모든 사용자가 **동일한 값**을 공유합니다.
* Service Center에서 앱 배포 없이 실시간으로 값을 수정할 수 있습니다.


* **시험 포인트:** "환경 설정값 관리", "배포 없이 수정 가능한 변수", "모든 사용자 공유".
* **사용 예시:**
* 외부 API 호출을 위한 Base URL.
* 이벤트 기간 동안만 보여줄 "할인율" 설정.
* 시스템 점검 중인지 확인하는 Boolean 플래그.



### 5. Session Variable (세션 변수)

* **주의:** 최신 **Reactive Web에서는 사용되지 않으며**, Traditional Web용 시험에서만 나옵니다.
* **특징:** 서버 세션에 저장되며 해당 사용자만 접근 가능합니다.
* **시험 포인트:** Reactive Web 시험이라면 "대신 Client Variable을 사용한다"는 점이 포인트입니다.

---

### 💡 시험 대비 요약표

| 변수 종류 | 저장 위치 | 유효 기간 | 데이터 타입 제한 | 핵심 키워드 |
| --- | --- | --- | --- | --- |
| **Local** | 메모리 | 해당 로직 실행 시 | 없음 | 임시 저장, 최소 범위 |
| **Input/Output** | 메모리 | 호출 시 전달 | 없음 | Interface, 로직 간 전달 |
| **Client Var** | 브라우저 | 삭제 전까지 | **Basic Type만 가능** | 사용자 환경 설정, Offline 가능 |
| **Site Prop** | 서버(DB) | 영구적 | **Basic Type만 가능** | 전역 설정, Service Center 수정 |

**시험 Tip:** 만약 문제에서 **"Complex Data Type(List나 Record)을 저장해야 한다"**고 하면 Client Variable이나 Site Property는 정답이 될 수 없습니다. 이때는 보통 Local Variable을 쓰거나 DB Entity에 저장해야 합니다.
